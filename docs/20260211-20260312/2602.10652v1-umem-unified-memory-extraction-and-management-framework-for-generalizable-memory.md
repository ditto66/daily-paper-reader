## 摘要
自演化记忆作为基于大语言模型（LLM）智能体的可训练参数，其提取（从经验中提炼见解）与管理（更新记忆库）必须紧密协调。现有方法主要优化记忆管理，而将记忆提取视为静态过程，导致泛化能力较差，使智能体积累了特定于实例的噪声而非鲁棒的记忆。为解决这一问题，我们提出了统一记忆提取与管理（UMEM），这是一个自演化智能体框架，通过联合优化大语言模型来同时提取和管理记忆。为了减轻对特定实例的过拟合，我们引入了语义邻域建模（Semantic Neighborhood Modeling），并通过 GRPO 利用邻域级边际效用奖励来优化模型。该方法通过评估语义相关查询簇中的记忆效用，确保了记忆的可泛化性。在五个基准测试上的广泛实验表明，UMEM 显著优于极具竞争力的基准方法，在多轮交互任务中实现了高达 10.67% 的提升。此外，UMEM 在持续演化过程中保持了单调增长曲线。代码和模型将公开发布。

## Abstract
Self-evolving memory serves as the trainable parameters for Large Language Models (LLMs)-based agents, where extraction (distilling insights from experience) and management (updating the memory bank) must be tightly coordinated. Existing methods predominately optimize memory management while treating memory extraction as a static process, resulting in poor generalization, where agents accumulate instance-specific noise rather than robust memories. To address this, we propose Unified Memory Extraction and Management (UMEM), a self-evolving agent framework that jointly optimizes a Large Language Model to simultaneous extract and manage memories. To mitigate overfitting to specific instances, we introduce Semantic Neighborhood Modeling and optimize the model with a neighborhood-level marginal utility reward via GRPO. This approach ensures memory generalizability by evaluating memory utility across clusters of semantically related queries. Extensive experiments across five benchmarks demonstrate that UMEM significantly outperforms highly competitive baselines, achieving up to a 10.67% improvement in multi-turn interactive tasks. Futhermore, UMEM maintains a monotonic growth curve during continuous evolution. Codes and models will be publicly released.

### 论文总结：UMEM (Unified Memory Extraction and Management Framework)

#### 1. 核心问题与整体含义
论文关注的是基于大语言模型（LLM）的**自进化智能体（Self-evolving Agents）**。
*   **研究动机**：现有的自进化智能体通常将“记忆提取”（从经验中提炼知识）视为静态的 Prompt 过程，而仅优化“记忆管理”（如何增删改）。这导致智能体容易陷入“死记硬背”的陷阱，积累大量针对特定实例的噪声，而非具有泛化性的原则，导致在面对新任务时表现不佳。
*   **核心问题**：如何协同优化记忆的提取与管理，以确保生成的记忆既能解决当前问题，又能泛化到语义相关的其他任务中。

#### 2. 方法论
UMEM 提出了一个统一的框架，将记忆提取和管理作为智能体的“外部参数”进行联合训练。
*   **核心思想**：模拟神经网络的优化过程，将智能体执行视为“前向传播”，将记忆优化器的更新视为“反向优化”。
*   **关键技术细节**：
    *   **语义邻域建模 (Semantic Neighborhood Modeling)**：在训练时，不只针对单个查询（Query）评估记忆，而是通过检索提炼出该查询的 $N$ 个语义近邻。只有当提取的记忆能同时提升整个邻域的性能时，才给予高奖励，从而抑制过拟合。
    *   **边际效用奖励 (Marginal Utility Reward)**：奖励函数由两部分组成：
        1.  **成功增益 (Success Gain)**：衡量记忆是否纠正了原本错误的预测。
        2.  **效率正则化 (Efficiency Regularization)**：在保证正确的前提下，奖励更简洁的推理步骤（减少 Token 消耗）。
    *   **GRPO 算法优化**：采用群体相对策略优化（Group Relative Policy Optimization），通过采样多组记忆更新动作并计算相对优势，实现端到端的策略训练。
    *   **在线记忆演化**：在训练过程中动态更新记忆库，强制模型学习如何管理不断变化的知识系统。

*   **消融实验**：详细探讨了“仅优化管理”、“仅优化提取”以及“邻域大小 $N$”对结果的影响。

#### 6 主要结论与发现
*   **性能提升**：UMEM 在多轮交互任务（ALFWorld）中相比基准提升了 **10.67%**，在复杂推理任务中也表现出显著优势。
*   **联合优化的必要性**：消融实验证明，单独优化记忆管理会导致性能大幅下降，记忆提取的质量是自进化的核心瓶颈。
*   **演化稳定性**：UMEM 在持续学习过程中表现出单调增长的性能曲线，有效避免了传统方法中常见的“记忆污染”和性能退化问题。
*   **效率增益**：UMEM 不仅提高了成功率，还显著减少了完成任务所需的平均步数。

#### 7. 优点
*   **泛化性强**：通过语义邻域建模，从根本上解决了记忆过拟合实例的问题。
*   **端到端协同**：首次实现了提取与管理的联合优化，解决了两者策略不匹配（Misalignment）的问题。
*   **即插即用**：训练好的 Mem-Optimizer 可以作为独立组件，为各种不同的冻结 LLM 提供记忆增强服务。

#### 8. 不足与局限
*   **训练数据依赖**：训练数据主要源自 MMLU 的采样，虽然在多个 Benchmark 上表现良好，但在极度冷门或垂直领域的泛化效果仍需进一步验证。
*   **计算开销**：虽然推理时高效，但训练阶段需要针对每个 Query 运行多次执行器以计算边际效用，训练成本相对较高。
*   **邻域敏感性**：实验显示邻域大小 $N$ 对结果有影响，如何自动确定最优邻域范围（而非人工设定 $N=3$）尚待研究。
