## 摘要
智能体多模态模型的强化学习通常面临交互崩溃（interaction collapse）的问题，即模型会学习减少工具使用和多轮推理，从而限制了智能体行为的优势。我们提出了 PyVision-RL，这是一个针对权重开放多模态模型的强化学习框架，旨在稳定训练并维持交互。我们的方法结合了过采样-过滤-排序（oversampling-filtering-ranking）的展开策略与累积工具奖励，以防止崩溃并鼓励多轮工具使用。利用统一的训练流水线，我们开发了用于图像和视频理解的 PyVision-Image 和 PyVision-Video。在视频推理方面，PyVision-Video 采用了按需上下文构建，在推理过程中选择性地采样与任务相关的帧，从而显著减少视觉 token 的使用。实验结果显示了强大的性能和更高的效率，证明了持续交互和按需视觉处理对于可扩展的多模态智能体至关重要。

## Abstract
Reinforcement learning for agentic multimodal models often suffers from interaction collapse, where models learn to reduce tool usage and multi-turn reasoning, limiting the benefits of agentic behavior. We introduce PyVision-RL, a reinforcement learning framework for open-weight multimodal models that stabilizes training and sustains interaction. Our approach combines an oversampling-filtering-ranking rollout strategy with an accumulative tool reward to prevent collapse and encourage multi-turn tool use. Using a unified training pipeline, we develop PyVision-Image and PyVision-Video for image and video understanding. For video reasoning, PyVision-Video employs on-demand context construction, selectively sampling task-relevant frames during reasoning to significantly reduce visual token usage. Experiments show strong performance and improved efficiency, demonstrating that sustained interaction and on-demand visual processing are critical for scalable multimodal agents.

### 1. 核心问题与整体含义（研究动机和背景）
论文旨在解决多模态大模型（MLLM）在作为智能体（Agent）进行强化学习（RL）时面临的**“交互崩溃”（Interaction Collapse）**问题。
*   **背景**：虽然 LLM 在文本领域通过 RL 实现了强大的推理能力，但在多模态领域，模型在 RL 微调后往往会学习到“偷懒”行为，即减少工具使用和多轮推理步骤，导致其无法充分利用外部工具（如 Python 解释器）来处理复杂的视觉任务。
*   **核心问题**：如何通过 RL 稳定地训练开源多模态模型，使其能够持续进行多轮交互、灵活调用工具，并提高处理长视频等任务的效率。

### 2. 方法论：核心思想与关键技术
PyVision-RL 提出了一个统一的智能体强化学习框架，核心包含以下三个关键组件：

*   **Python 作为原始工具（Dynamic Tooling）**：模型不使用预定义的静态工具集，而是直接生成 Python 代码。在图像任务中用于裁剪、缩放、像素分析；在视频任务中用于动态抽帧。
*   **累积工具奖励（Accumulative Tool Reward）**：
    *   为了防止模型减少工具调用，设计了公式：$R = R_{acc} + 0.1 \cdot n_{tc} \cdot \mathbb{1}\{R_{acc}=1\}$。
    *   其中 $R_{acc}$ 是答案正确性奖励（0或1），$n_{tc}$ 是工具调用次数。只有在答案正确的前提下，才会奖励更多的工具使用，从而鼓励深度推理而非无效堆砌。
*   **过采样-过滤-排序（Oversampling-Filtering-Ranking）策略**：
    *   **过采样**：生成比训练批次更多的 Rollout 样本。
    *   **过滤**：剔除代码运行崩溃、超时或无奖励方差的无效轨迹。
    *   **排序（标准差排序）**：根据组内奖励的标准差进行排序，优先选择难度适中（既有正确也有错误）的样本进行训练，类似于课程学习。
*   **视频按需上下文构建（On-demand Context Construction）**：
    *   针对视频任务，不预先进行均匀采样，而是将完整视频仅加载到 Python 运行环境。模型根据需求编写代码主动抓取特定帧，极大降低了视觉 Token 的消耗。
