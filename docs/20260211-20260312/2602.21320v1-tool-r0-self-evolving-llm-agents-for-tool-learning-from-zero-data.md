## 摘要
大语言模型（LLMs）正成为能够利用工具解决复杂任务的自主智能体（autonomous agents）的基础。强化学习（RL）已成为注入此类智能体能力的常用方法，但通常是在严格受控的训练设置下进行的。它往往依赖于精心构建的任务-解决方案对和大量的人类监督，这为通往超智能系统的开放式自进化制造了根本性障碍。在本文中，我们提出了 Tool-R0 框架，在零数据假设下，通过自我博弈强化学习（self-play RL）从零开始训练通用工具调用智能体。Tool-R0 从同一个基础 LLM 初始化，通过互补奖励共同进化生成器（Generator）和求解器（Solver）：前者在后者的能力边界提出有针对性的挑战性任务，而后者则学习通过真实的工具调用来解决这些任务。这创建了一个无需预先存在任务或数据集的自进化循环。在不同工具使用基准测试上的评估表明，Tool-R0 相比基础模型实现了 92.5% 的相对提升，并在相同设置下超过了全监督工具调用基准。我们的工作通过分析协同进化、课程动态和缩放行为，进一步为自我博弈 LLM 智能体提供了实证见解。

## Abstract
Large language models (LLMs) are becoming the foundation for autonomous agents that can use tools to solve complex tasks. Reinforcement learning (RL) has emerged as a common approach for injecting such agentic capabilities, but typically under tightly controlled training setups. It often depends on carefully constructed task-solution pairs and substantial human supervision, which creates a fundamental obstacle to open-ended self-evolution toward superintelligent systems. In this paper, we propose Tool-R0 framework for training general purpose tool-calling agents from scratch with self-play RL, under a zero-data assumption. Initialized from the same base LLM, Tool-R0 co-evolves a Generator and a Solver with complementary rewards: one proposes targeted challenging tasks at the other's competence frontier and the other learns to solve them with real-world tool calls. This creates a self-evolving cycle that requires no pre-existing tasks or datasets. Evaluation on different tool-use benchmarks show that Tool-R0 yields 92.5 relative improvement over the base model and surpasses fully supervised tool-calling baselines under the same setting. Our work further provides empirical insights into self-play LLM agents by analyzing co-evolution, curriculum dynamics, and scaling behavior.

### 2. 方法论：核心思想与关键技术
Tool-R0 框架的核心是**生成器（Generator）**与**求解器（Solver）**的协同进化循环：
*   **核心思想**：初始化两个相同的 LLM 角色。Generator 负责合成具有挑战性的工具调用任务，Solver 学习解决这些任务。两者通过互补的奖励信号共同成长。
*   **关键技术细节**：
    *   **接地任务规范（Grounded Task Specification）**：为了防止生成内容坍塌，Generator 接收轻量级的域配置（如金融、医疗等）作为引导，生成包含用户请求、工具菜单和金标答案的任务。
    *   **Generator 奖励设计**：
        *   **格式奖励 ($r_{fmt}$)**：确保输出符合 JSON 格式和特定标签。
        *   **有效性奖励 ($r_{valid}$)**：验证金标答案中的工具是否在菜单中，参数是否完整，且参数值是否出现在问题中。
        *   **课程奖励 ($r_{curr}$)**：这是核心创新。利用 Solver 的答案不确定性设计**难度引导奖励**。它像一个“带通滤波器”，奖励那些 Solver 胜率在 25%-75% 之间的任务（即处于 Solver 的能力边界），太简单或太难的任务奖励较低。
    *   **Solver 训练**：在 Generator 生成的经过过滤和排序（由易到难）的课程数据集上，使用 **GRPO（群体相对策略优化）** 算法进行训练，奖励基于工具调用的准确性。
    *   **协同进化**：每一轮迭代后，Solver 变强，Generator 随之调整难度，提出更难的任务。

### 6. 主要结论与发现
*   **零数据胜过全监督**：Tool-R0 在 1.5B 模型上实现了 92.5% 的相对提升，且性能超越了使用 21 万条人工标注数据训练的基线。
*   **自生成课程的优越性**：模型生成的任务分布比静态的人工数据集更广、更均衡，能更精准地针对模型自身的薄弱环节。
*   **角色分离至关重要**：如果 Generator 和 Solver 共享参数，由于奖励目标冲突（探索 vs. 精确执行），性能会大幅下降（下降约 36%）。
*   **能力边界驱动**：有效的学习需要 Generator 始终在 Solver 的“能力边界”出题，平滑的高斯难度衰减比硬性的难度阈值更稳定。

### 7. 优点
*   **范式突破**：证明了在工具学习领域，完全摆脱人类数据是可行的，且效果更好。
*   **算法创新**：设计了精巧的难度感知奖励函数（Band-pass reward），解决了自博弈中任务难度失控的问题。
*   **实用性强**：Tool-R0 可以作为一种“中途训练”手段，显著增强模型在后续微调中的表现。

### 8. 不足与局限
*   **算力信息缺失**：未明确标注具体的 GPU 型号和总训练时长，不便于复现成本评估。
*   **评估成本**：课程奖励需要对 Solver 进行多次采样（K=8）来估计难度，这在训练大规模模型时会带来显著的计算开销。
*   **知识边界限制**：由于 Generator 和 Solver 源自同一模型，其生成的任务仍受限于模型预训练时积累的潜在知识，难以凭空产生模型完全未见过的全新领域知识。
*   **奖励作弊风险**：在极小规模模型（如 0.5B）上，仍观察到少量“奖励作弊”现象，即生成看似正确但逻辑有瑕疵的任务。
