## 摘要
基于大语言模型的 Web 智能体在自动化网页任务方面展现出了巨大的潜力。然而，由于预测环境变化的能力有限，当前的 Web 智能体难以推理出合理的动作，且可能缺乏对执行风险的全面意识，从而过早地执行导致损失和任务失败的风险动作。为了应对这些挑战，我们提出了 WAC，这是一种集成了模型协作、后果模拟和反馈驱动动作优化的 Web 智能体。为了克服单个模型的认知孤立，我们引入了一种多智能体协作流程，使动作模型能够咨询作为网页环境专家的世界模型以获取战略指导；随后，动作模型将这些建议落实为可执行的动作，利用环境状态转移动态的先验知识来增强候选动作的提议。为了实现具备风险意识的弹性任务执行，我们引入了一个两阶段演绎链。专门负责环境状态转移的世界模型模拟动作结果，随后由评判模型进行审查，并在必要时触发动作修正反馈。实验表明，WAC 在 VisualWebArena 和 Online-Mind2Web 上分别实现了 1.8% 和 1.3% 的绝对增益。

## Abstract
Web agents based on large language models have demonstrated promising capability in automating web tasks. However, current web agents struggle to reason out sensible actions due to the limitations of predicting environment changes, and might not possess comprehensive awareness of execution risks, prematurely performing risky actions that cause losses and lead to task failure. To address these challenges, we propose WAC, a web agent that integrates model collaboration, consequence simulation, and feedback-driven action refinement. To overcome the cognitive isolation of individual models, we introduce a multi-agent collaboration process that enables an action model to consult a world model as a web-environment expert for strategic guidance; the action model then grounds these suggestions into executable actions, leveraging prior knowledge of environmental state transition dynamics to enhance candidate action proposal. To achieve risk-aware resilient task execution, we introduce a two-stage deduction chain. A world model, specialized in environmental state transitions, simulates action outcomes, which a judge model then scrutinizes to trigger action corrective feedback when necessary. Experiments show that WAC achieves absolute gains of 1.8% on VisualWebArena and 1.3% on Online-Mind2Web.

*   **研究背景**：基于大语言模型（LLM）的 Web 智能体在自动化网页任务中表现出潜力，但仍面临两大挑战：
    *   **认知孤立（C1）**：单模型决策受限于其知识覆盖面，难以准确预测复杂的环境状态转换。
    *   **缺乏风险意识（C2）**：现有智能体通常在提出动作后直接执行，缺乏对潜在后果的预判，一旦执行错误动作（如误删、误购），往往导致任务不可逆的失败。
*   **核心问题**：如何提升 Web 智能体在复杂、动态网页环境中的决策合理性，并建立一种在执行前识别并修正风险动作的机制。

### 2. 方法论
WAC 框架将决策过程分为两个阶段，通过多模型协作实现“先思后行”：
*   **阶段 I：动作生成（Action Generation）**
    *   **按需协作（Collaboration-on-demand）**：引入一个路由模块（Router），根据当前观察判断是否需要咨询“世界模型”（作为环境专家）。若需要，世界模型提供高层战略指导。
    *   **动作提议**：动作模型结合环境观察、历史记录和世界模型的指导，生成多个候选动作。
*   **阶段 II：动作推演（Action Deduction）**
    *   **后果模拟**：世界模型针对每个候选动作模拟执行后的网页状态（自然语言描述）。
    *   **风险评估与反馈**：评判模型（Judge Model）对模拟结果进行打分（0, 0.5, 1）。
    *   **闭环修正（Action Correction）**：若所有候选动作得分均低于阈值，则将评判模型的反馈（理由）传回动作模型进行重新生成，形成“提议-模拟-评价-修正”的闭环，直到产生高置信度动作或达到最大迭代次数。

### 6. 主要结论与发现
*   **性能提升**：WAC 在 VisualWebArena 上达到 24.5% 的成功率（比基线提升 1.8% 绝对值），在 Online-Mind2Web 上达到 16.0%（提升 1.3% 绝对值）。
*   **修正机制的价值**：实验证明，在执行前进行模拟和修正能有效避免早期错误累积，尤其是在任务的第一步。
*   **反馈可替代采样**：发现反馈驱动的修正机制在某种程度上可以替代大规模的随机动作采样，从而在减少采样多样性的同时保持高性能。

### 7. 优点
*   **闭环纠错能力**：不同于以往只做“选择”的智能体，WAC 具备“重写”和“修正”动作的能力。
*   **按需协作**：通过 Router 机制平衡了推理成本与决策质量，避免了在简单步骤中过度消耗算力。
*   **免微调特性**：作为一种即插即用的框架，能够快速适配各种强大的视觉语言模型（VLM）。

### 8. 不足与局限
*   **模型依赖性**：性能高度依赖于底层 VLM 的感知和推理能力，若基础模型对网页 DOM 理解有误，WAC 难以完全弥补。
*   **模拟幻觉风险**：世界模型对环境的模拟是近似的，可能存在幻觉，导致错误的反馈信号。
*   **推理延迟**：由于引入了多轮“模拟-评价-修正”循环，单步决策的时间开销显著高于传统的 ReAct 模式。
*   **长程模拟限制**：目前模拟步数仅为 1 步，对于需要多步预判才能发现风险的极复杂任务，效果可能受限。
